## 模块说明

### Main Process

应用程序主进程，负责：

- 窗口生命周期管理（主面板窗口、浏览器窗口）
- IPC 消息路由与处理
- 原生模块加载与协调
- 快捷键注册与分发
- 应用配置持久化

### Renderer

渲染进程，负责用户界面：

- 设置面板：快捷键映射、透明度、缩放、收藏夹等配置界面
- 浏览器窗口：嵌入网页内容（默认 B 站），支持导航与控制

### Native Module

C++ 原生模块，通过 node-gyp 编译为 Node.js 插件：

- 键盘钩子模块：注册系统级低级键盘钩子（WH_KEYBOARD_LL），在全屏游戏中截获快捷键
- UIAccess 置顶模块：通过 UIAccess API + SetWinEventHook，确保浏览器窗口在全屏 DirectX 游戏上保持置顶

### Storage

基于 electron-store 的持久化方案，数据以 JSON 文件存储在用户配置目录。

管理：窗口位置、快捷键映射、透明度、收藏夹、缩放级别等设置。

### IPC

基于 Electron IPC 的通信层。

- preload.js 提供白名单桥接
- Renderer 通过受控通道发送请求
- Main Process 处理并响应
- 详见 IPC.md

### UI

双窗口模型：

- 主面板窗口（固定大小、非置顶）
- 浏览器窗口（可调大小、支持置顶与透明度）

### Future AI Layer

预留 AI 集成层，未来用于：

- 调用 LLM API 实现视频摘要、内容推荐
- 本地上下文引擎分析用户行为
- AI 攻略生成与推送
